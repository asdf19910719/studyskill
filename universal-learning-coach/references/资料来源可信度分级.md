# 资料来源可信度分级

## P0：官方资料

官方文档、官方教程、官方源码文档和官方 API 参考。用于建立事实基础和定义边界。

## P1：权威资料

官方博客、标准文档、权威书籍、核心维护者文章。用于补充背景、设计动机和实践细节。

## P2：高质量补充资料

系统系列博客、深度教程、课程讲义、会议演讲。用于辅助理解和补充案例，但要和 P0/P1 交叉验证。

## P3：社区讨论

StackOverflow、Reddit、论坛、Issue 讨论。用于观察真实问题、排障线索和常见坑，不作为核心事实来源。

## P4：碎片资料

短视频、搬运文章、无来源摘要。只用于启发理解，不作为事实依据。

## 使用原则

- 结论性定义优先引用 P0。
- 工程经验可参考 P1/P2，但要说明适用版本和场景。
- P3/P4 只能辅助，不得替代官方资料。
- 资料过旧或无来源时必须标注不确定性。
