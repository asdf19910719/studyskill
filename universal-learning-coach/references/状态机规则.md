# 状态机规则

## 目标

用短命令驱动连续学习流程。用户只需要输入 `$universal-learning-coach 初始化`、`继续`、`学完了`、`结束`、`复习`，Skill 必须读取状态文件决定下一步，不要求用户重复长提示词。

## 状态文件优先级

1. 优先读取 `learning_state.json`。
2. 如果不存在，再读取 `_学习状态.md`。
3. 如果两者都不存在，进入 `未初始化 / 初始化`。
4. 如果 JSON 与 Markdown 冲突，以 JSON 为准，并建议同步 Markdown。

## 状态字段

`flow_status` / `当前流程状态` 可选值：

- `未初始化`
- `资料诊断中`
- `概念盘点完成`
- `需要扩展资料`
- `学习中`
- `待考试`
- `待更新`
- `待复习`
- `已完成`

`next_action` / `下一步动作` 可选值：

- `初始化`
- `诊断资料`
- `概念预热`
- `扩展笔记`
- `继续学习`
- `开始考试`
- `结束更新`
- `生成复习计划`

## 短命令映射

| 用户命令 | 读取状态后执行 |
| --- | --- |
| `初始化` | 创建状态文件，扫描资料，生成知识地图，逐概念诊断，生成学习路线 |
| `继续` | 根据 `next_action` 选择扩展、学习、考试或复习 |
| `学完了` | 进入考试模式，一次只问一题 |
| `结束` | 追加更新学习状态、错题本、复习卡片，并设置下一步 |
| `复习` | 生成今日复习计划 |

## 状态转移

```text
未初始化 / 初始化
→ 资料诊断中 / 诊断资料
→ 概念盘点完成 / 概念预热
→ 需要扩展资料 / 扩展笔记
→ 学习中 / 继续学习
→ 待考试 / 开始考试
→ 待更新 / 结束更新
→ 待复习 / 生成复习计划
→ 学习中 / 继续学习
```

## `继续` 决策顺序

1. 如果 `next_action` 是 `概念预热`，先对 `current_concept` 做短讲解：概念是什么、解决什么问题、和当前主题有什么关系，再给最小自测。
2. 如果 `next_action` 是 `扩展笔记`，先诊断和扩展资料。
3. 如果 `next_action` 是 `开始考试`，直接出一题并等待回答。
4. 如果 `next_action` 是 `结束更新`，输出 Obsidian 更新内容。
5. 如果 `next_action` 是 `生成复习计划`，先生成复习计划。
6. 如果没有特殊动作，生成一个 30-60 分钟学习任务。

## 输出要求

每次结束时都明确给出应写回的状态：

```markdown
## 当前流程状态
学习中

## 下一步动作
继续学习
```

如果工具允许写文件，更新 `learning_state.json` 和 `_学习状态.md`；否则输出可复制内容并说明未写入。
